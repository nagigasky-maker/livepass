# EXPASS / LIVEPASS — ローンチ戦略まとめ

> 最終更新: 2026-04-28

---

## プラン構成

| プラン | 価格 | 主な制限・特典 |
|--------|------|----------------|
| **FREE** | 無料 | EVENT/WORKSHOP/EXHIBITION 合計3件まで、チケット販売不可、PASS販売不可 |
| **STANDARD** | 未設定 | 投稿無制限、チケット販売可、PASS販売可（10件まで）、PRO限定レイアウト不可 |
| **PRO** | 未設定 | STANDARD+、PASSコンテンツ無制限、シネマレイアウト等全解放 |
| **BUSINESS** | 未設定 | 招待コード経由でのみ付与 |

## ローンチ時に公開できる機能（完成済み）

### コンテンツ作成
- **SCREEN フィード** — 記事・イベント・ワークショップ・展示・レコードのタイムライン
- **ARTICLE 作成** — 複数レイアウト（ポスター/スクエア/ジン等）、AIアシスト入力補完、カテゴリタグ
- **EVENT 作成** — カバー画像、チケット価格設定（STANDARD+）、チーム共同クレジット
- **WORKSHOP 作成** — イベントと同構成、ワークショップ特化メタデータ
- **EXHIBITION 作成** — ビジュアルアーティスト向け展示登録
- **RECORD 作成** — レコードジャケットカード、レーベルデザイン、SCREENフィード連携
- **ATELIER / Record Booth** — ブラウザ内DJレコーディング → レコード投稿

### 発見・検索
- **SEARCH** — アカウント検索（Firestore連携）、フォロー/ディスカバリー/フォロー中タブ
- **CALENDAR** — イベントカレンダー表示、AIコンシェルジュ検索、カテゴリフィルター
- **AI コンシェルジュ** — アーティスト情報応答、「フォローをする」「ページをみる」ボタン

### アカウント・ソーシャル
- **サインアップ/オンボーディング** — ユーザー名・メール・パスワード、利用規約同意、Firestore保存
- **プロフィール** — カバー・アバター・ロール表示、フォロー/フォロワー数
- **フォロワー管理** — フォロー/フォロワーリスト表示・操作
- **DM/メッセージ** — 1:1チャット（localStorageベース）、通知センター
- **招待コード** — コード経由でプランアップグレード
- **NFC/QR カードクレーム** — アーティストカード収集

### チケット・スキャン
- **チケット予約** — Stripeシートによる支払い（STANDARD+のみ）
- **QRコード生成** — チケット保有者向け `expass://t/<sessionId>`
- **会場スキャン** — jsQRでQR読み取り → Firestoreチェックイン更新

### 設定・管理
- **設定画面** — プロフィール編集、アバター/カバークロップ、プラン選択、メール認証、データエクスポート
- **プランゲーティング** — `livepass_plan.js` による全ページ統一ゲート（FREE 3件制限、チケット販売制御）

## ローンチまでに必須の作業

### 🔴 ローンチブロッカー（必須）

1. **プランアップグレード決済**
   - STANDARD/PRO への Stripe Subscription 接続
   - 現状はプランが選べるだけで課金されない

2. **PASS商品登録ダッシュボード完成**
   - EXPASSの核心機能
   - 別セッションで進行中だが完成が必要

3. **チケット決済のサーバーサイド検証**
   - クライアント側処理のみで本番運用は不可
   - Stripe API server-side 実装が必要

4. **メール認証フローの確認**
   - 未認証ユーザーの投稿・購入ブロックの動作確認

5. **Firestoreセキュリティルール確認**
   - 他ユーザーのデータを読み書きできないか検証

### 🟡 ローンチ前推奨（品質向上）

6. **プラン料金表ページ**
   - 料金比較ランディングページが存在しない

7. **スキャン画面のカメラ拒否時対応**
   - 権限拒否時のフォールバックUI未実装

8. **PASS購入→QR→スキャン 一気通貫テスト**
   - PASSダッシュボード完成後に実施

9. **PWA / Service Worker設定**
   - オフラインキャッシュ未設定（App Store提出予定がある場合は必須）

10. **`/api/redeem-invite` エンドポイント**
    - 招待コード検証がクライアント側プレフィックス判定のみ

### 🟢 v1.1以降でOK

- REVENUE ダッシュボード（売上可視化）
- STORE / 物販カタログ
- Frame Studio
- DM Firestore リアルタイム同期
- フォロワー Firestore 連携
- カードクレーム APIエンドポイント
